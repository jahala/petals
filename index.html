Self-audit

The agent checks its own work

Before a human sees it. Before PR review. Before it ships.

C
Colors against colors.md. Hex codes checked against the palette. Data URIs excluded. Nearest-match suggested for every off-palette value.
T
Typography against typography.md. Font-family, weight, size, line-height verified against the type scale. Heading/body classification checked.
S
Spacing against the grid. px values scanned. Off-scale values flagged as warnings with closest valid alternatives.
V
Voice against voice.md. Forbidden terms detected case-insensitive. Tone rules checked. Replacement suggested for every flagged term.

Architecture

Files, not dashboards

No server. No database. No SaaS. Just files and git.

.brand/ DESIGN.md 9-section design system colors.md palette, semantic roles, CSS vars typography.md font families, type scale, spacing voice.md forbidden terms, tone, examples identity.md name, tagline, mission, story assets/ logo.svg hero.jpg

.brand/ is a cache, not source code.

Gitignored. The project commits .petalrc instead — a small config that records the source URL, version, and overrides.

Progressive disclosure.

~160 tokens at discovery. ~5,700 at activation. Individual files loaded on demand. Most brand work under 7,000 tokens.

Distributed via git.

One central repo. Projects fetch via shallow clone or HTTPS. Drift detection at session start. One push propagates to every connected project.

Design

Built for how agents actually work

Finite context budget

Agents have ~200K tokens. Brand files use progressive disclosure — the agent loads only what it needs, when it needs it. Most brand tasks take under 7,000 tokens.

Generic defaults

Without brand context, every agent produces Tailwind blue + Inter + neutral tone. Petals gives the agent a target to aim at — explicit, machine-readable, discoverable.

File-based truth

Brand rules live in markdown files — reviewable in PRs, diffable in git, readable by humans and agents alike. No API. No UI. No vendor lock-in.
