The problem

The brand lives in a beautiful PDF. The agent has never seen it.

A company spends months on its identity. None of it reaches the agent writing the UI — so it reaches for the defaults, and someone catches it later.

Someone types "build a pricing page." The agent has no reason not to pick the framework's default blue. It uses Inter because that is what shipped with the template. It writes a headline that sounds like every other landing page on the internet.

The brand team's primary color is a warm clay. Headings are meant to be Fraunces. The voice guide forbids the exact hype word the agent just used. The agent knew none of this — the brand was a 40-page PDF it was never handed.

So it surfaces in PR review, for the fourth time this week. Someone leaves a comment. Someone else pastes the hex codes into the next prompt. Three months on, a campaign page ships in last year's palette because that was the only stylesheet the agent could find.

How it works

Three skills: extract, distribute, check

One pipeline turns brand assets into files agents read natively — then keeps every project in sync, and audits the output before a human sees it.

/petal init

Extract

Point petals at a folder of brand assets. The agent reads PDFs, markdown, images, and JSON with its native capabilities — writes only the explicit values, flags anything missing rather than guessing it.

/petal init --from

Distribute

Connect projects to one central brand repo. Drift detection warns when the central brand moves ahead. Updates flow through while project-level overrides are preserved. One push, every project syncs.

/petal check

Check

The agent audits its own output against the rules before a human reviews it. Color, typography, spacing, and voice — four dimensions, caught at generation time, with the nearest on-brand value suggested for each miss.

Self-audit

The agent checks its own work

Before a human sees it. Before review. Before it ships.

C
Color, against colors.md

Every hex matched to the palette. Data URIs and base64 excluded. The nearest palette color named for each off-brand value.

T
Type, against typography.md

Heading and body families checked against the hierarchy. Body text in a heading font is an error, not a warning.

S
Spacing, against the 4px grid

Padding, margin, and gap values scanned. Off-scale values flagged as warnings with the closest valid step.

V
Voice, against voice.md

Forbidden terms caught case-insensitive. Each one named with the preferred replacement from the guide.

Honest scope

What petals is for — and what it isn't

petals is a design assist for agents — it informs how the agent designs inside your brand before a line is generated, and audits what came out after. It is not a design editor, and it will not invent your brand for you.

Built for
  • Teams shipping UI and copy with coding agents in the loop.
  • Projects that already have brand guidelines — a PDF, a palette, a voice guide.
  • Keeping many projects on one central brand, with project overrides where needed.
  • Catching off-brand color, type, and voice at generation time, not in review.
Not for
  • Inventing a brand from scratch — petals reads what exists, it does not design.
  • Pixel-perfect visual QA — it checks rules in files, not rendered layouts.
  • Replacing designers — the brand is theirs; petals carries their decisions.
  • Guessing missing values — a missing hex is flagged for a human, never approximated.
Architecture

Files, not dashboards

No server. No database. No account. The brand lives in markdown, distributed with git.

.brand/ 
.brand/
  DESIGN.md      # 9-section design system
  colors.md      # palette, roles, CSS vars
  typography.md  # families, scale, spacing
  voice.md       # forbidden terms, tone
  identity.md    # name, tagline, story
  assets/
    logo.svg     # byte-for-byte original

.brand/ is a cache, not source.

Gitignored. The project commits .petalrc instead — a small config recording the source repo, version, and any overrides.

Read on demand.

Files load individually, only when the work needs them — colors for UI, voice for copy. The agent treats the brand like a tsconfig: present by default, applied without being asked.

Lightweight by design.

Most brand work lands under 7K tokens — progressive disclosure keeps the agent's context for the work, not the rules.
